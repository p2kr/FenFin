# Expense Tracker

A full-stack personal finance expense tracker designed for real-world conditions (unreliable networks, browser refreshes, retries).

## How to Run

Requirements: Docker and Docker Compose.

1. Clone the repository.
2. Run `docker compose up --build`
3. Access the frontend at `http://localhost:3000` (Note: Frontend is mapped to port 3000 in compose to avoid clash with backend, though both can be handled behind nginx if preferred)

*Backend API is available at `http://localhost:8080`*

## Local Development (Without Docker)

If you prefer to run the services directly on your host machine:

### Backend
1. Ensure you have a local PostgreSQL instance running and update the `application.yml` database credentials.
2. Navigate to the `backend` directory and run: `./mvnw spring-boot:run`

### Frontend
1. Navigate to the `frontend` directory.
2. Install dependencies using Yarn: `yarn install`
3. Start the development server: `yarn dev`
4. Access the UI at `http://localhost:5173` (Vite's default port).

## Key Design Decisions

1. **Money representation**: Used `BigDecimal` in Java and `NUMERIC(19,4)` in PostgreSQL to avoid floating point inaccuracies. In JSON, amounts are transmitted as Strings to preserve precision.
2. **Idempotency Strategy**: 
   - A unique `Idempotency-Key` (UUID) is generated by the frontend form upon mount or successful submission.
   - The backend tracks the `idempotency_key` and a `request_hash` (SHA-256 of the canonical request body).
   - If the same request is submitted again (due to retry or double click), the backend returns the already created record.
   - If a new request uses an existing key, a `409 Conflict` is returned.
3. **Single Source of Truth for Totals**: The server computes the total expenses based on the applied filters, ensuring clients always agree.
4. **Resilient Frontend**: The frontend implements retries with backoff for transient failures while reusing the idempotency key.

## Trade-offs and Omissions

- **No Authentication**: Omitted to fit within a timebox.
- **Single-User Scope**: All expenses belong to a global list.
- **Minimal Styling**: Tailwind CSS was used for a clean but functional design.
- **No Pagination**: Fetching all items for simplicity.
- **No Edits/Deletes**: Out of scope for this assignment.
